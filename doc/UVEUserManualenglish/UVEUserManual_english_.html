UVE

User manual

UVE is a tool to design and generate the structure of a testbench (TB) based on SystemVerilog (SV) and the Universal Verification Methodology (UVM). UVE stands for Unified Verification Environment and includes a graphical user interface, a code generator, compilation scripts and a library of verification IPs (VIP).

Requested environment

UVE itself runs on Windows and Unix. To be able to simulate the generated TB the Questa® Simulator[1] version 10.1 or newer from Mentor Graphics®[2] is needed.

Starting with UVE

The main goal of UVE is to help you build your testbench from scratch. With the toplevel of your design and some knowledge about the UVM methodology you’re ready to go.

Quick Step

New Project

A wizard guides you through creating a new project. You always have to fill in all mandatory fields before continuing to the next step, but you can always go back to modify entered data.

Main Window

Upon launching UVE the main window is empty; only the toolbars are visible. The heart of the main window is a text editor. Each file opened will be displayed here in a separate tab. In addition the following elements display themselves when needed.

  1. Project Files         Tree view of all files used in the current project
  2. Project View         Structural view of the project
  3. Project Wide Search         Search text in all files of the current project
  4. Simulation Output         Output of the simulation of the current project

The View menu allows to control the visibility of all these elements.
